# Code Review: Swagger DTO 类型化重构

## 变更概览

对所有 NestJS controller 进行系统性重构，主要变更：

1. **添加 Swagger 装饰器** — 所有 controller 方法添加 `@ApiTags`、`@ApiOperation`、`@ApiOkResponse`、`@ApiBearerAuth`
2. **DTO 类型安全化** — 将 `@Body() data: any` 替换为具体的 Zod DTO 类型（使用 `nestjs-zod` 的 `createZodDto`）
3. **新增 50+ DTO 文件** — 按模块组织为 `input.dto.ts` 和 `response.dto.ts`
4. **OAuth2 逻辑修复** — `oauth2.controller.ts` 中增加了 `code` 空值校验和 userinfo 响应结构改进

## 变更统计

- **修改文件**: 60 个 controller/service 文件
- **新增文件**: 50+ DTO 文件（跨 auth、cms、infra、mall、member、mp、pay、system 模块）

## 审查发现

### ✅ 正面评价

1. **一致性好** — 所有 DTO 统一使用 Zod schema + `createZodDto` 模式，风格一致
2. **文档完善** — 每个字段都有 `.describe()` 中文描述，Swagger UI 友好
3. **类型安全** — 消除了所有 `any` 类型的 `@Body()` 参数，改为强类型 DTO
4. **模块组织清晰** — 输入/响应 DTO 分离，按模块归类到 `dto/` 目录
5. **嵌套结构正确** — `UserResponseDto`、`MemberResponseDto` 等正确处理了关联关系

### ⚠️ 需要关注的问题

#### 1. `SuccessResponseDto` 跨模块复用

`auth-response.dto.ts` 中定义的 `SuccessResponseDto` 被 10+ 个模块导入使用：
```typescript
import { SuccessResponseDto } from '../../auth/dto/auth-response.dto';
```
**建议**: 考虑将通用 DTO 提取到 `shared/dto/` 或 `common/dto/` 目录，避免跨模块依赖。当前可行但不够理想。

#### 2. `oauth2.controller.ts` 行为变更

- `authorize` 方法的 `@Query()` 参数从独立参数变为 `OAuth2AuthorizeQueryDto`，但实际行为不变（只是解构方式不同）
- `token` 方法增加了 `code` 空值校验（`oauth2.controller.ts:106-108`），这是一个**功能性修复**
- `userinfo` 返回值增加了 `sub` 字段（`oauth2.controller.ts:170`），**可能影响前端调用方**

#### 3. `user.service.ts` 类型放宽

`updateProfile` 的 `data` 参数类型从 `string` 改为 `string | null`（`user.service.ts:184`），这是正确的修复，与 DTO 定义一致。

#### 4. `@ApiOkResponse` 用于删除操作

多个 controller 的 `remove` 方法使用 `@ApiOkResponse({ type: ...ResponseDto })`，但删除操作通常返回空或简单成功消息，不一定返回完整资源对象。需确认 service 层实际返回值是否匹配。

#### 5. `notify-template.controller.ts` 的 `sendTest` 方法

原代码从 `@Body()` 中分别提取 `userId`、`templateCode`、`variables`，重构后使用 `SendTestNotifyDto`。逻辑一致，但 `variables` 的默认值处理从 `|| {}` 移到了调用处（`body.variables || {}`），行为不变。

#### 6. 部分 DTO 使用 `z.any()`

`mall-response.dto.ts` 中 `properties`、`sliderPicUrls` 等字段使用了 `z.any()`：
```typescript
properties: z.any().describe('商品属性规格 JSON 数据'),
```
这在 Swagger 文档中会显示为任意类型，降低文档质量。建议用 `z.record(z.string(), z.any())` 或更具体的类型。

### 🔍 建议验证

1. **编译检查** — 运行 `npm run build` 确保所有新增 DTO 和类型修改无编译错误
2. **Swagger UI 验证** — 启动服务后访问 `/api` 确认文档正确渲染
3. **回归测试** — 重点测试 OAuth2 流程（authorize → token → userinfo）和用户资料修改
